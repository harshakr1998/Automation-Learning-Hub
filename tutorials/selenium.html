Comprehensive Selenium Guide (Java)

Introduction to Selenium

Selenium is a suite of tools for automating web browsers. It supports various programming languages, including Java.

  • Components of Selenium:
    • Selenium WebDriver
    • Selenium IDE
    • Selenium Grid

Setting Up Selenium with Java

  1. Install Java (JDK) and an IDE (e.g., Eclipse, IntelliJ IDEA).
  2. Download Selenium JAR files or add Maven dependencies.
  3. Add WebDriver executables (e.g., ChromeDriver).
<dependency>
    <groupId>org.seleniumhq.selenium</groupId>
    <artifactId>selenium-java</artifactId>
    <version>4.x.x</version>
</dependency>

Writing Your First Selenium Script

import org.openqa.selenium.WebDriver;
import org.openqa.selenium.chrome.ChromeDriver;

public class FirstSeleniumTest {
    public static void main(String[] args) {
        System.setProperty("webdriver.chrome.driver", "path/to/chromedriver");
        WebDriver driver = new ChromeDriver();
        driver.get("https://www.example.com");
        System.out.println("Page title is: " + driver.getTitle());
        driver.quit();
    }
}

Locators in Selenium

Selenium provides multiple ways to locate elements:

  • ID: driver.findElement(By.id("id"));
  • Name: driver.findElement(By.name("name"));
  • CSS Selector: driver.findElement(By.cssSelector("css"));
  • XPath: driver.findElement(By.xpath("//xpath"));

Page Object Model (POM)

The Page Object Model (POM) is a design pattern that enhances test maintainability by separating UI elements from test logic. 

public class LoginPage {
    WebDriver driver;
    By username = By.id("username");
    By password = By.id("password");
    By loginButton = By.id("login");

    public LoginPage(WebDriver driver) {
        this.driver = driver;
    }

    public void login(String user, String pass) {
        driver.findElement(username).sendKeys(user);
        driver.findElement(password).sendKeys(pass);
        driver.findElement(loginButton).click();
    }
}

Selenium Grid

Selenium Grid allows distributed test execution across multiple browsers and machines.

  • Start a hub using: java -jar selenium-server-standalone.jar -role hub
  • Register a node using: java -jar selenium-server-standalone.jar -role node -hub http://localhost:4444/grid/register

Headless Browser Testing

Run tests without a visible browser to improve performance.

ChromeOptions options = new ChromeOptions();
options.addArguments("--headless");
WebDriver driver = new ChromeDriver(options);
driver.get("https://www.example.com");
System.out.println(driver.getTitle());
driver.quit();

Frequently Asked Questions

  • Can Selenium handle CAPTCHAs? No, CAPTCHAs require manual or third-party solutions.
  • Can Selenium be used for performance testing? Not directly; use JMeter or LoadRunner for such tasks.
  • What browsers are supported? Chrome, Firefox, Edge, Safari, etc.
